# 第三阶段本地资料、历史与语音输入设计

设计日期：2026年7月19日  
状态：已完成交互确认，待用户复核书面规格

## 1. 目标

在不依赖学校统一身份认证、教务系统或服务端语音识别的前提下，完成第三阶段的本地体验版：

- 提供年级、专业和当前目标设置。
- 根据年级提供差异化的学业与生涯成长入口。
- 在当前设备恢复非心理模块的对话历史。
- 提供可降级的语音转文字输入。

本阶段不实现独立注册登录，不模拟学校账号，不显示无真实接口支撑的办理进度。

## 2. 方案选择

采用纯前端本地体验方案：

- 使用`localStorage`保存本地学生资料和非心理模块历史。
- 使用`SpeechRecognition`或`webkitSpeechRecognition`实现语音转文字。
- 不上传、不保存音频文件。
- 不支持语音识别时自动隐藏或禁用语音入口，文本输入仍可完整使用。

不采用`IndexedDB`和服务端音频上传。当前数据量不需要更复杂的存储层，音频上传还会引入额外的授权、留存、删除和隐私管理问题。

## 3. 功能范围

### 3.1 本地学生资料

“我的”页增加资料设置，字段包括：

- 年级：大一、大二、大三、大四。
- 专业：可选的纯文本字段。
- 当前目标：可选的纯文本字段。

不收集姓名、学号、手机号、身份证号等可识别身份信息。页面固定显示“资料仅保存在当前设备，不等于学校账号”。

### 3.2 年级化学业与生涯成长

| 年级 | 版本 | 服务主题 |
|---|---|---|
| 大一 | 探索版 | 专业认知、兴趣与优势探索、大学目标制定、校园资源推荐、生涯人物访谈 |
| 大二 | 定位版 | 职业方向探索、能力差距分析、竞赛与项目建议、考研考公就业初步比较、学期能力提升计划 |
| 大三 | 行动版 | 实习推荐与准备、简历素材积累、岗位能力训练、考研院校与方向分析、模拟面试 |
| 大四 | 冲刺版 | 招聘信息分析、简历精准修改、面试训练、升学复试、Offer与就业选择、未就业学生重点帮扶 |

选择年级后，成长模块的欢迎文案、版本标识和快捷问题同步更新。未设置年级时使用现有通用入口，不推测学生年级。

### 3.3 本地历史记录

持久化范围：

- 校园生活。
- 学业与生涯成长。
- 事务办理。

心理陪伴历史始终仅保留在当前内存会话。页面内切换模块时可恢复；刷新、关闭页面或重新打开后不恢复。

“我的”页按模块展示本地历史摘要，支持：

- 恢复指定模块对话。
- 删除指定模块历史。
- 二次确认后清除全部非心理历史。

删除提示明确说明操作只影响当前设备。

### 3.4 语音转文字

对话输入区增加语音按钮。操作流程为：

1. 学生点击语音按钮。
2. 浏览器申请麦克风权限并开始识别。
3. 界面显示“正在听”状态，再次点击可停止。
4. 识别结果填入文本输入框。
5. 学生可修改文字，确认后手动发送。

识别结果不自动发送。不支持语音识别、权限被拒绝、未检测到语音或发生网络错误时，向用户提供针对性提示，不生成空消息。

## 4. 模块边界

### `js/storage.js`

负责：

- 本地存储键名与数据版本。
- 资料与历史记录的读取、校验、限制、保存和删除。
- 损坏数据或不兼容版本的安全回退。

不负责DOM渲染和对话调用。

### `js/profile.js`

负责：

- 年级、专业和当前目标表单。
- 字段长度与可选值校验。
- 年级到成长版本、主题和快捷问题的映射。

不负责身份认证。

### `js/voice.js`

负责：

- 语音识别能力检测。
- 识别的启动、停止、结果和错误映射。
- 把识别文字交给输入框，但不触发发送。

不负责录音文件存储和服务端上传。

### `js/chat.js`

保留页面导航、场景会话、回答反馈和API调用。通过上述模块读取资料、持久化允许保存的会话，并接收语音转写结果。

## 5. 数据模型与限制

本地数据采用带版本的单一根对象：

```json
{
  "version": 1,
  "profile": {
    "grade": "freshman",
    "major": "",
    "goal": ""
  },
  "sessions": {
    "0": [],
    "1": [],
    "3": []
  }
}
```

约束：

- `grade`只允许`freshman`、`sophomore`、`junior`、`senior`或空值。
- `major`最多40个字符。
- `goal`最多120个字符。
- 每个可持久化模块最多保留40条消息。
- 单条用户文本或AI回答的持久化纯文本最多8000个字符。
- 本地数据中不允许出现心理模块键`2`。

页面输出使用`textContent`或安全的表单值，不把本地资料作为HTML注入。

## 6. 数据流

```text
本地资料
  → 选择年级
  → 更新成长版本与快捷问题
  → 用户选择主题或语音输入
  → 文本确认
  → 发送给AI
  → 非心理模块保存到当前设备

心理陪伴
  → 只保存在当前内存会话
  → 不写入localStorage
  → 刷新或重新打开后清空
```

## 7. 异常处理

- 存储内容不是合法JSON、根对象结构不正确或版本不支持时，回退到空资料和空历史，不阻断应用启动。
- 写入超出浏览器容量时，保留当前内存会话，告知用户未能保存到当前设备。
- 语音错误分为不支持、权限拒绝、未检测到语音、网络错误和其他失败。
- 语音识别失败时不清除输入框原有文字，不触发消息发送。
- 清除全部历史需要二次确认，取消后不改变任何数据。

## 8. 测试与验收

### 自动化测试

- 资料字段的可选值、长度和损坏数据回退。
- 只保存场景0、1、3，拒绝持久化场景2。
- 每个场景40条消息的上限。
- 大一至大四与四套生涯主题的正确映射。
- 语音能力不存在时的降级、语音结果只写入输入框不自动发送。
- 个人资料、历史、语音按钮和隐私说明的页面结构。

### 浏览器验收

在375×812视口检查：

- 本地资料表单、年级选择和保存反馈。
- 历史恢复、单模块删除和全部清除确认。
- 成长模块随年级更新版本标识和快捷问题。
- 语音启动、停止、结果回填和不支持时的文本降级。
- 刷新后非心理历史恢复，心理历史不恢复。

## 9. 上线边界

- `localStorage`不是跨设备账户系统，清理浏览器数据后无法恢复。
- 浏览器内置语音识别的可用性、语言支持和数据处理方式受浏览器及系统服务影响，页面需要明确说明。
- 真实注册登录、统一身份认证、办理进度和跨设备历史需等待学校提供相应平台与接口。
