# 爱·合师AI辅导员

面向合肥师范学院学生的移动端AI辅导员原型。当前版本提供校园生活、学业与生涯成长、心理陪伴和事务办理四类对话服务。

## 运行要求

- Node.js 22及以上版本。
- 可用的科大讯飞星火4.0Ultra应用凭据。

## 本地启动

1. 安装依赖：

   ```bash
   npm install
   ```

2. 复制环境变量模板并填写真实凭据：

   ```bash
   cp .env.example .env
   ```

3. 启动服务：

   ```bash
   npm start
   ```

4. 浏览器访问：<http://localhost:3001>

可通过`PORT`环境变量修改监听端口。真实`.env`文件不得提交，也不得放入可公开访问的目录。

## 测试

运行全部自动化测试：

```bash
npm test
```

当前共132项自动化测试。

测试覆盖以下边界：

- 环境变量、服务端源码和项目配置不能通过HTTP访问。
- 聊天参数、消息长度、请求类型和请求体大小受到限制。
- 网页抓取拒绝本机、私网、链路本地地址和非HTTP协议。
- 北京时间的日期、星期和时间保持一致。
- 前后端使用显式的`role`和`content`传递对话历史。
- 心理危机表达触发明确支持信息和可拨打的求助资源。
- 学生手册检索能命中制度名称、手册页码和PDF页码。
- 事务类问题在调用模型前由服务端强制预检索手册。
- 首页、四模块、个人中心和两级回答反馈具有结构测试。
- 本地资料、非心理历史、跨页面写入冲突和心理会话隔离具有可执行测试。
- 大一至大四的生涯阶段映射、资料请求边界和语音转写降级具有可执行测试。

## 安全边界

- 服务器只公开`index.html`、`manifest.json`以及`assets/`、`css/`、`js/`下的前端资源。
- `/api/chat`仅接受`application/json`，请求体上限为256KB；对话历史最多20条，单条最多8000字符。
- `fetch_url`只允许访问公开HTTP或HTTPS网页，拒绝本机、内网、URL凭据和自动重定向，并限制响应类型与大小。
- 模型错误不会向浏览器返回内部异常或环境变量内容。
- 校园场所、事务流程、联系方式和时间节点必须先核验官方来源；无法核验时明确说明，不使用推测内容兜底。

## 学生手册索引

本地制度依据来自`references/2023合师学生手册内文 (1).pdf`。需先安装包含`pdftotext`的Poppler，然后运行：

```bash
npm run build:handbook
```

命令会生成`data/student_handbook.json`。该索引只供服务端检索，静态服务不会公开`data/`目录。事务办理以及命中制度关键词的校园、学业问题，会在模型回答前强制预检索手册，避免模型跳过工具。

手册是2023年7月版本，不能自动视为当前有效规定。回答应显示制度名称、手册页码和PDF页码；时间、名额、金额、入口和联系方式必须核验学校最新正式通知。

## 本地资料、历史与语音

- “我的”页可保存年级、专业和当前目标；资料保存在当前浏览器的`localStorage`，存储键为`aihesh.local.v2`。
- 选择“暂不选择”并清空专业、目标后重新保存，可清空本地资料。“清空全部”只删除校园、成长和事务历史，不删除资料；清除浏览器中本站的站点数据可一并删除两者。
- 校园生活、学业与生涯成长、事务办理的历史仅保存在当前设备。心理对话不写入本地历史，刷新或关闭页面后不恢复；为生成当次回答，心理对话仍会发送至本应用服务端和配置的AI服务。
- 成长模块提问时，年级、专业和目标会随当次请求发送给AI；其他三个模块不携带这些资料。
- 本地历史是明文浏览器存储，不是加密数据库。在共用设备上使用后，应主动删除或清空历史，不要在资料和普通对话中填写密码、密钥等敏感信息。
- 语音输入依赖浏览器或操作系统的语音识别能力，可能联网并由相关服务处理音频。本应用不保存音频；转写文本只进入输入框，用户确认或编辑后才发送。不支持语音识别时，可继续使用文本输入。

## 当前产品边界

- 每条正常AI回答支持“已解决/未解决”反馈；首次未解决会请求针对性补充，第二次仍未解决时生成可复制的人工转介摘要。
- 学校尚未提供真实在线人工渠道，因此当前不会伪造工单、值班状态或响应时间。
- “我的”页已提供当前设备的本地资料、会话统计、历史恢复和清除。
- 当前仍是本地体验版：未接入真实注册登录或统一身份认证，未实现服务端及跨设备历史，未对接学校办事系统的办理进度，也未接入真实人工工单。
